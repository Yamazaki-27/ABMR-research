## 1. 配線

ハードウェアマニュアルに従って配線する。

### 電源ライン

- コントローラのハードウェアマニュアル通り、**1a/1b のスイッチを用意すること**
- **boot信号は必須**（電源とbootが分かれているのはリチウムイオン電池の立ち上げ時間を考慮するため）

### CAN通信

| 項目 | 内容 |
|------|------|
| 配線方式 | Hi/Lo をデイジーチェーン接続 |
| 終端抵抗 | **必須** |
| CANGND | 接続しなくても動作可 |

### EM-OUT（非常停止出力）

- モータドライバへ非常停止信号を送る端子
- CAN経由でも同等機能あり（非常停止 → ドライバへ駆動禁止指令）
- IO接続の方がCAN通信より信頼性が高いため、安全要件の高い用途に有効
- **繋がなくても動作は可能**

### Lidar

- **電源ケーブル** と **イーサネットケーブル** のみでOK

## 2. Roboshop立ち上げ

### PC接続

1. イーサネットケーブルでPCとコントローラを**直接接続**
2. コントローラ上部バーに表示されているIPアドレスを確認
3. PCのIPアドレスを**手動設定**し、同一ネットワークに入れる
4. オンラインマニュアルの**クイックスタートガイド**に従って接続

### 基本操作

- ロボットを認識したら**ダブルクリック**で詳細設定へ
- 画面右下にエラー数が表示される → **すべてのエラーを解消することが目標**
- 不要な機能は左メニューのチェックを外すとすっきりする（「全て」選択で無効機能も表示）

### Push / Pull の概念

GitHubと同様の考え方：

```
Pull  ← ロボットの現在設定をPC側へ読み出し
Push  → PC側の設定をロボットへ書き込み
```

> **正しい手順**: まず `Pull` で現設定を取得 → 変更 → `Push` で書き込み

### トリガ機能（デフォルト無効）

- **デフォルトでは無効** になっているため注意
- バンパー入力時の非常停止など安全機能の実現には**必須**

## 3. Lidar立ち上げ

### 初期設定

1. 専用ソフトウェアを使用し、PCとイーサネットで直接接続
2. トップメニューから該当モデルをクリックして専用ツールを起動

> ⚠️ **注意**: Windowsのセキュリティに引っかかる場合、最初は各機種の `.exe` を直接起動してセキュリティを無効化する必要あり

### IPアドレス設定

コントローラと同一ネットワークに入れるようIPアドレスを設定する。

```
例: 192.168.1.88
```

（※元メモ記載は `192.186.1.88` だが `192.168.1.88` が正しい可能性あり。マニュアルで確認）

### 設定手順

1. オンラインマニュアル「チュートリアル → ハードウェア → Lidarの設定」を参照
2. 設定値を変更して書き込み
3. Roboshopに戻り「Laser」項目からIPアドレス・モデル名を選択
4. 接続成功すればLidar入力状況が画面で確認可能になる

## 4. モータ立ち上げ

### 事前確認項目

設定前に以下の仕様を把握しておくこと：

| 確認項目 | 内容 |
|----------|------|
| CAN通信仕様 | CAN-Open / CAN-FD など |
| ギヤ比 | モータ仕様書より |
| エンコーダ種別 | インクリメンタル / アブソリュート |
| 1回転あたりパルス数 | マニュアル値でOK（キャリブレーションで補正） |

### PC接続方法

CAN または RS485 経由（USB変換器が必要）

- CAN変換器：高価 + プロトコルによる不具合リスクあり
- **RS485変換器を推奨**（安価で安定）
- デバッグ用に**端子台へRS485ラインを引き出しておくこと**

### RS485 配線の注意

RS485は2線式（送受信を切り替えて通信）。機器によって呼称が異なるため注意：

| 表記A | 表記B | 意味 |
|-------|-------|------|
| RS485(+) | RS485 A | 同じ（＋側） |
| RS485(−) | RS485 B | 同じ（−側） |

> RX/TX の表記に惑わされないこと。A/+ 同士、B/− 同士を接続する。

### 設定手順

1. オンラインマニュアル「チュートリアル → ハードウェア → Kinko（iシリーズ）モータ設定」を参照
2. 設定値を書き込んだら**ソフトを再起動**し、値が正しく変わったか再確認
3. Roboshopに戻りエンコーダ設定等を実施

### よくあるトラブルと対処

| トラブル | 対処 |
|----------|------|
| Roboshopでのモータ型式が違う | 正しいモデルを選択し直す |
| CAN通信仕様の設定ミス（例：CANFD → CANOpen） | 仕様を確認して修正 |
| 設定修正後もタイムアウトエラーが残る | モータ用ソフトで強制エラーリセット、またはコントローラを含めた全体再起動 |

> ✅ **正常確認**: モータが励磁されると**高周波音（ジー音）** が聞こえるようになる

## 6. キャリブレーション

### 事前動作確認

1. 駆動モータを**浮かせた状態**でキーボード操作
2. W / A / D キーで前進・左右旋回方向が正しいか確認
3. 問題なければ床面に下ろす

### キャリブレーション手順

クイックスタートガイドに従い「事前マッピング」から実行：

- 部屋を**大きく1周（円を描くように）** すればOK
- 自動的に円弧・前後動作を繰り返してデータを補正
- 計算上の動きと実際の動きを合わせ込む
- 許容範囲に入るまで**最大5回**繰り返す

### キャリブレーションNGの場合

ハードウェア的な問題を疑う：

- 駆動2輪の向きがずれている
- 補助輪が効きすぎて駆動輪が浮き気味になっている
- モータ・コントローラ・Lidarが床面と平行でない
- 車輪径の設定値が間違っている
- トレッドの設定値が間違っている

## 7. その他・補足

### LEDライト制御

**DMX512** 通信対応のLEDライトはコントローラから制御可能。

> ⚠️ 国産品はないため、中国からの購入が必要

### Wi-Fiルーター

自律走行にはWi-Fi経由でPCと接続する必要がある。**家庭用ルーターは不適**。

| メーカー | 備考 |
|----------|------|
| TP-Link | 日本国内で流通、産業用あり |
| MOXA | 産業用Wi-Fiルーター |

### コントローラ内蔵Hotspot機能

- SEERコントローラには `@mobi` と同様のHotspot機能あり
- **日本の技適に未対応のため、デフォルトで無効化**
- 有効化すれば使用可能だが **自己責任**
